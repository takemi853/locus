### セットアップ完了 (22:36)

**Context:** claude-memory-compiler のセットアップと動作確認

**Key Exchanges:**
- Claude Code のhooksを使って会話を自動でナレッジベースに蓄積するシステムを構築
- `~/.claude/settings.json` にグローバルhooksを設定（SessionStart / PreCompact / SessionEnd）
- SessionStart hookが `session-start.py` を呼び出してナレッジベースのインデックスと最新ログをコンテキストに注入する仕組み

**Decisions Made:**
- グローバル設定に絶対パスでhooksを登録 → どのプロジェクトでも動作する
- プロジェクトローカルの `.claude/settings.json` にはhooksを入れない → 重複発火を防ぐ
- RAGなし・インデックスファイルだけで検索する → 個人スケール（50〜500記事）ではLLMがインデックスを読む方がベクトル検索より精度が高い（Karpathyの知見）

**Lessons Learned:**
- グローバルhooksとプロジェクトローカルhooksが両方存在すると同じセッションで2回発火する
- Claude Agent SDK は bundled CLI (`_bundled/claude`) を使って非同期でエージェントを実行する
- 初回フラッシュはセットアップ直後でコンテキストが空のため失敗することがある。2回目以降は正常動作
- `uv run python scripts/compile.py` で手動コンパイルできる
- 18時以降のセッション終了時に自動でコンパイルがトリガーされる

**System Architecture:**
```
会話 → SessionEnd/PreCompact hook → flush.py → daily/YYYY-MM-DD.md
    → compile.py → knowledge/ に記事生成
        → SessionStart hook → 次回会話にコンテキスト注入
```

### Session (22:50)

**Context:** User asked for a concise review of important gotchas when configuring Claude Code hooks.

**Key Exchanges:**
- User asked about things to watch out for with hooks configuration
- Assistant summarized 4 key pitfalls: duplicate firing, infinite recursion with Agent SDK, PreCompact `transcript_path` bug, and first-flush failure

**Lessons Learned:**
- Global (`~/.claude/settings.json`) and project-local (`.claude/settings.json`) hooks for the same event fire **twice** in the same session — only register in one place
- Hooks that invoke Claude Agent SDK can cause infinite recursion (SessionEnd → SDK → new Claude session → SessionEnd → ...); break the loop with an env var guard (`CLAUDE_INVOKED_BY`)
- Known Claude Code bug (#13668): `transcript_path` can be empty in PreCompact events during long sessions — always guard against empty path in hook scripts
- First flush after initial setup often fails due to empty context; expected behavior, resolves on second invocation
